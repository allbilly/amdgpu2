# MGPUSim V5 Code Object Investigation Report

## Executive Summary

This document summarizes our investigation into MGPUSim's support for V5 HSA code objects, generated by modern ROCm (v7.x).

## Key Findings

### 1. MGPUSim V5 Code Object Parsing
- **Location**: `mgpusim/amd/insts/hsaco.go`
- **Support Status**: ✅ Parser exists and parses V5 kernel descriptors
- **CodeObjectV5** type is defined
- `parseV5KernelDescriptor()` function handles V5 format

### 2. MGPUSim Emulator Limitations
- **Location**: `mgpusim/amd/emu/`
- **Issue**: GCN3 emulator doesn't support all GFX900 (Vega) instructions
- **Affected Instructions**:
  - `v_add_co_u32_e32` (VOP3C)
  - `v_addc_co_u32_e32` (VOP3C)
  - Other VOP3C/VOP3 instructions

### 3. ROCm Code Object Versions
| Version | ROCm Support | Notes |
|---------|-------------|-------|
| V2 | ❌ Deprecated | Not supported by ROCm 7.x |
| V3 | ⚠️ Legacy | ROCm 5.x can generate but internally uses V4+ |
| V4 | ✅ Current | Default for older GPUs |
| V5 | ✅ Current | Default for CDNA/RDNA |
| V6 | ✅ Current | Default for newer GPUs |

### 4. Code Object Format Comparison
```
V3 Header (256 bytes):
  - magic "HSA CodeObject"
  - version (3)
  - HSA Runtime Version

V5 Header (64 bytes):
  - version (5)
  - vendor "AMD"
  - arch "AMDGPU"
  - kernel descriptor follows
```

## Solutions Explored

### Solution 1: Use Older ROCm (5.6)
```bash
podman pull rocm/dev-ubuntu-22.04:5.6
hipcc --offload-arch=gfx900 -mcode-object-version=3 shaders/vectoradd.cl -o vectoradd.hsaco
```
**Result**: ROCm internally generates V4+ format despite flag

### Solution 2: Use MGPUSim with Existing Kernels
- Use pre-compiled kernels from `amd/benchmarks/amdappsdk/vectoradd/`
- These kernels use simpler instruction sets
- Pass verification with correct KernelArgs struct

### Solution 3: Run on Real GPU via PM4
- Use `amdgpu-poc` with raw shader binaries
- PM4 packets directly program the GPU
- No HSA runtime required


## Recommendations for Full V5 Support

1. **Update GCN3 Emulator**: Add VOP3C instruction support
2. **Add gfx900 Target**: Create a separate emulator for Vega architecture
3. **Fix Kernarg Marshaling**: V5 kernels expect 296-byte kernarg segments
4. **Use HSA Runtime**: For testing V5 code objects on real GPU
